How It Works

Scope Detection

When a REXX interpreter is created, it automatically searches the DOM for an ancestor element with the RexxScript CSS class:

  • If found: Functions are registered to that DOM element (isolated scope)
  • If not found: Functions are registered to the global window object (legacy behavior)

Script Container Structure

<div class="RexxScript">
    <!-- Functions registered here in: element.__rexxFunctions -->
</div>

<div>
    <!-- Functions registered here in: window -->
</div>

JavaScript API

// Create interpreter with explicit scope
const container = document.getElementById('script1-container');
const interpreter = new RexxInterpreter(null, {
    scopeElement: container,
    output: console.log
});

// Or auto-detect RexxScript class
const interpreter2 = new RexxInterpreter();
// Will find RexxScript class automatically

Architecture

Scope Registry System

Each interpreter maintains a scopeRegistry with methods to manage function registration:

Method Purpose
registerFunction(name, func) Register function to scope
getFunction(name) Retrieve function from scope
hasFunction(name) Check if function exists
registerMetadata(name, meta) Register function metadata
getMetadata(name) Retrieve function metadata

Storage Locations

  • Isolated Scope: domElement.__rexxFunctions and domElement.__rexxMetadata
  • Global Scope: window.FUNCTION_NAME and window.METADATA_NAME

The current implementation isolates function registration. In the next phase, we'll also isolate variable pools so that:

  • Variables don't leak between scripts

Use Cases

  • Multi-tenant Applications: Each tenant's scripts run in isolation
  • Plugin Systems: Multiple plugins can safely coexist
  • Dynamic Scripting: User-uploaded scripts don't interfere with each other
  • Testing Frameworks: Each test runs in isolated scope
  • Dashboard Applications: Multiple independent widgets with embedded REXX
  • Collaborative Tools: Different users' scripts work in parallel without conflicts
